GetIndexFromTable

GetIndexFromTable, which takes a manufacturerNum as an input and returns a string. Here's a quick explanation of what the code does:

  • The method is designed to search for a particular row in a table called mdlTable where the manufacturerNum matches values in either column 10 or column 11.
  • mdlTable is assumed to be a DataTable, and the AsEnumerable() method converts it into an Enumerable that allows the use of LINQ operations.
  • The code uses LINQ to query the rows in the table:
    • It uses .FirstOrDefault() to find the first row where either column 10 or column 11 matches the provided manufacturerNum.
    • If a match is found, the expression .Field<string>(0) is used to get the value of the first column (index 0) in the matching row.
    • If no matching row is found, it returns an empty string (string.Empty) as a default value.
private string GetIndexFromTable(string manufacturerNum)
        {
            // Return the index (first column) where manufacturer number is found in column 10 or 11
            return mdlTable.AsEnumerable()
                           .FirstOrDefault(row => row.Field<string>(10) == manufacturerNum || row.Field<string>(11) == manufacturerNum)?
                           .Field<string>(0) ?? string.Empty;
        }

PerformMatching

The PerformMatching() method matches manufacturer numbers and returns three lists: exact matches, fuzzy matches, and unmatched manufacturer numbers.

  • Data Preparation:

    • Prepares data structures (HashSet and dictionaries) to collect unique manufacturer numbers and create relevant mappings.

  • Data Collection:

    • Calls CollectManufacturerNumbers() to gather and organize manufacturer numbers and create mappings.

  • Matching:

    • Performs exact and fuzzy matching using PerformExactAndFuzzyMatching(), populating exactMatchedList and fuzzyMatchedList.

  • Identify Unmatched Items:

    • Calls FindUnmatchedManufacturerNumbers() to populate the list of unmatched items.

  • Return:

    • Returns a tuple of the three lists: exactMatchedList, fuzzyMatchedList, and unmatchedManufacturerNumsList, all sorted by index.
public (List<MaximoData> exactMatchedList, List<MaximoData> fuzzyMatchedList, List<Tuple<string, string>> unmatchedManufacturerNumsList) PerformMatching()
        {
            // Prepare hash sets and dictionaries to track manufacturer numbers and their indices
            var manufacturerNums = new HashSet<string>(); // Stores unique manufacturer numbers
            var indexToManufacturerMap = new Dictionary<string, HashSet<string>>(); // Maps indices to manufacturer numbers
            var manufacturerNumToIndexMap = new Dictionary<string, string>(); // Maps manufacturer numbers to their corresponding indices

            // Collect all unique manufacturer numbers and map them
            CollectManufacturerNumbers(manufacturerNums, indexToManufacturerMap, manufacturerNumToIndexMap);

            // Lists to store exact and fuzzy matches
            var exactMatchedList = new List<MaximoData>();
            var fuzzyMatchedList = new List<MaximoData>();
            var unmatchedManufacturerNumsList = new List<Tuple<string, string>>(); // Stores unmatched manufacturer numbers

            // Perform exact and fuzzy matching
            PerformExactAndFuzzyMatching(manufacturerNums, exactMatchedList, fuzzyMatchedList);

            // Find and list all unmatched manufacturer numbers
            FindUnmatchedManufacturerNumbers(manufacturerNums, exactMatchedList, fuzzyMatchedList, unmatchedManufacturerNumsList, manufacturerNumToIndexMap);

            // Return the results sorted by index
            return (exactMatchedList.OrderBy(x => x.INDEX).ToList(), fuzzyMatchedList.OrderBy(x => x.INDEX).ToList(), unmatchedManufacturerNumsList);
        }

            manufacturerNumToIndexMap[colValue] = index; // Map manufacturer number to its index

            Debug.WriteLine($"Manufacturer number {colValue} is already associated with a different index ({manufacturerNumToIndexMap[colValue]}), attempting to associate with index {index}.");

            Debug.WriteLine($"Manufacturer number {colValue} is already associated with index {index}.");

private void PerformExactAndFuzzyMatching(HashSet<string> manufacturerNums, List<MaximoData> exactMatchedList, List<MaximoData> fuzzyMatchedList)

        var matchedItems = new Dictionary<MaximoData, string>(); // Tracks whether an item was exactly or fuzzily matched

## PerformExactMatching
The `PerformExactMatching()` method checks if an item (`MaximoData`) has an exact match with manufacturer numbers from a given set (`manufacturerNums`). Heres a concise summary:

- **Loop Through Manufacturer Numbers**:    
    - Iterates through each manufacturer number in the `manufacturerNums` set.

- **Check for Exact Match**:    
    - Checks if either of the fields (`CXMANUFACTURERNUM` or `CXMANTYPED`) of `item` match the current manufacturer number.
    - If a match is found, it retrieves the `INDEX` from an external table using `GetIndexFromTable()`.

- **Update Lists and Mappings**:    
    - If an index is found (`indexResult` is not null or empty):
        - Updates the `item` with the retrieved index.
        - Adds the item to the `exactMatchedList`.
        - Marks the item as "exactly matched" in the `matchedItems` dictionary.
        - Adds the manufacturer number to the `matchedManufacturerNums` set.
    - Logs the match for debugging and returns `true` to indicate a successful match.

- **Return No Match**:    
    - If no exact match is found, the method returns `false`.


~~~csharp 
private bool PerformExactMatching(MaximoData item, HashSet<string> manufacturerNums, Dictionary<MaximoData, string> matchedItems, HashSet<string> matchedManufacturerNums, List<MaximoData> exactMatchedList)
        {
            foreach (var manufacturerNum in manufacturerNums)
            {
                // Check if the item has an exact match in either CXMANUFACTURERNUM or CXMANTYPED fields
                if ((item.CXMANUFACTURERNUM != null && item.CXMANUFACTURERNUM.Equals(manufacturerNum)) ||
                    (item.CXMANTYPED != null && item.CXMANTYPED.Equals(manufacturerNum)))
                {
                    var indexResult = GetIndexFromTable(manufacturerNum); // Get index from table
                    if (!string.IsNullOrEmpty(indexResult))
                    {
                        item.INDEX = indexResult; // Assign index to the item
                        exactMatchedList.Add(item); // Add the item to the exact match list
                        matchedItems[item] = "Exact"; // Mark item as exactly matched
                        matchedManufacturerNums.Add(manufacturerNum); // Add manufacturer number to matched list
                        Debug.WriteLine($"MATCHER: Exact match found in MaximoData: {item.CXMANUFACTURERNUM} or {item.CXMANTYPED}, matched with MDL: {manufacturerNum}, assigned INDEX: {item.INDEX}");
                        return true; // Exit loop as soon as a match is found
                    }
                }
            }
            return false; // No exact match was found
        }

## PerformSubstringAndFuzzyMatching
The `PerformSubstringAndFuzzyMatching()` method attempts to find a match for an item (`MaximoData`) using substring and fuzzy matching techniques if no exact match is found. Heres a brief explanation:

- **Skip If Already Exactly Matched**:    
    - If `item` is already marked as an exact match in `matchedItems`, the method returns to avoid unnecessary processing.

- **Perform Substring Matching**:    
    - Calls `PerformNormalSubstringMatching()` to try to match `item` based on normal substring matching.
    - If a substring match is found, the `item` is added to the `fuzzyMatchedList`.

- **Bidirectional Substring Matching**:    
    - If the normal substring match fails (`isSubstringMatch` is `false`), it attempts **bidirectional substring matching** by calling `PerformBidirectionalSubstringMatching()`.
    - This approach tries matching from both ends (i.e., checking if either the manufacturer number contains the item's value or vice versa).

- **Fuzzy Matching**:    
    - If neither of the substring methods successfully matched the `item`, it calls `PerformFuzzyMatching()`.
    - This step is more flexible and tries to find matches with some level of difference or error tolerance.

- **Order of Operations**:    
    - The matching techniques are applied in this order: **Normal Substring Matching**  **Bidirectional Substring Matching**  **Fuzzy Matching**. The item is added to `fuzzyMatchedList` if a match is found in any of these stages.

~~~~csharp 
private void PerformSubstringAndFuzzyMatching(MaximoData item, HashSet<string> manufacturerNums, Dictionary<MaximoData, string> matchedItems, HashSet<string> matchedManufacturerNums, List<MaximoData> fuzzyMatchedList)
        {
            // Skip already exactly matched items to prevent overriding
            if (matchedItems.ContainsKey(item) && matchedItems[item] == "Exact")
            {
                return;
            }

            // First, try normal substring matching
            bool isSubstringMatch = PerformNormalSubstringMatching(item, manufacturerNums, matchedItems, matchedManufacturerNums, fuzzyMatchedList);

            // If no substring match, attempt bidirectional substring matching
            if (!isSubstringMatch)
            {
                PerformBidirectionalSubstringMatching(item, manufacturerNums, matchedItems, matchedManufacturerNums, fuzzyMatchedList);
            }

            // If neither substring match worked, perform fuzzy matching
            if (!matchedItems.ContainsKey(item))
            {
                PerformFuzzyMatching(item, manufacturerNums, matchedItems, matchedManufacturerNums, fuzzyMatchedList);
            }
        }

PerformNormalSubstringMatching

The PerformNormalSubstringMatching() method attempts to find an exact match or substring match for an item (MaximoData) against a set of manufacturer numbers (manufacturerNums). Heres a concise explanation:

  • Initialize Matching Flag:

    • Sets isSubstringMatch to false initially to track if a substring match occurs.

  • Loop Through Manufacturer Numbers:

    • Iterates through each manufacturer number in manufacturerNums.

  • Exact Match Check:

    • Checks if the item's fields (CXMANUFACTURERNUM or CXMANTYPED) have an exact match with the manufacturer number.
    • If an exact match is found:
      • Retrieves the INDEX using GetIndexFromTable().
      • If INDEX is not empty, it:
        • Marks the item as exactly matched (matchedItems and matchedManufacturerNums).
        • Logs the match and returns true.

  • Substring Match Check:

    • If no exact match is found, the method proceeds to substring matching.
    • Checks if the item's fields (CXMANUFACTURERNUM or CXMANTYPED) contain the manufacturer number (case-insensitive).
    • If a match is found:
      • Assigns an INDEX using GetIndexFromTable().
      • If INDEX is not empty:
        • Adds the item to the fuzzyMatchedList.
        • Marks the item as a fuzzy match in matchedItems.
        • Adds the manufacturer number and the matched item's manufacturer number to matchedManufacturerNums.
        • Logs the match, sets isSubstringMatch to true, and breaks the loop.

  • Return Match Status:

    • Returns isSubstringMatch to indicate whether a substring match was successfully made.
private bool PerformNormalSubstringMatching(MaximoData item, HashSet<string> manufacturerNums, Dictionary<MaximoData, string> matchedItems, HashSet<string> matchedManufacturerNums, List<MaximoData> fuzzyMatchedList)
{
    bool isSubstringMatch = false;

    foreach (var manufacturerNum in manufacturerNums)
    {
        bool exactMatchFound = false;

        // Check for exact match or substring match in CXMANUFACTURERNUM and CXMANTYPED fields
        if ((item.CXMANUFACTURERNUM != null && manufacturerNum.Equals(item.CXMANUFACTURERNUM)) ||
            (item.CXMANTYPED != null && manufacturerNum.Equals(item.CXMANTYPED)))
        {
            item.INDEX = GetIndexFromTable(manufacturerNum); // Get the index for the exact match

            if (!string.IsNullOrEmpty(item.INDEX))
            {
                //exactMatchedList.Add(item); // Add to exact match list
                matchedItems[item] = "Exact"; // Mark the item as exactly matched
                matchedManufacturerNums.Add(manufacturerNum); // Add manufacturer number to matched list
                Debug.WriteLine($"MATCHER: Exact match found in MaximoData: {item.CXMANUFACTURERNUM} or {item.CXMANTYPED}, matched with MDL: {manufacturerNum}, assigned INDEX: {item.INDEX}");
                exactMatchFound = true;
                return true; // Exit loop as soon as an exact match is found
            }
        }

        if (exactMatchFound)
            break; // Exit if exact match was found

        // Perform normal substring match for manufacturer number
        if ((item.CXMANUFACTURERNUM != null && item.CXMANUFACTURERNUM.ToLower().Contains(manufacturerNum)) ||
            (item.CXMANTYPED != null && item.CXMANTYPED.ToLower().Contains(manufacturerNum)))
        {
            if (!matchedItems.ContainsKey(item))
            {
                item.INDEX = GetIndexFromTable(manufacturerNum); // Assign index for substring match
            }

            if (!string.IsNullOrEmpty(item.INDEX))
            {
                fuzzyMatchedList.Add(item); // Add to fuzzy matched list
                matchedItems[item] = "Fuzzy"; // Mark as substring matched (considered fuzzy)

                // Add both the substring and the complete string as matched
                matchedManufacturerNums.Add(manufacturerNum);
                matchedManufacturerNums.Add(item.CXMANUFACTURERNUM);

                Debug.WriteLine($"MATCHER: Substring match found in MaximoData: {item.CXMANUFACTURERNUM} or {item.CXMANTYPED}, matched with MDL: {manufacturerNum}, assigned INDEX: {item.INDEX}");

                isSubstringMatch = true;
                break; // Exit loop as soon as a substring match is found
            }
        }
    }

    return isSubstringMatch; // Return if substring match was successful
}

PerformBidirectionalSubstringMatching

The PerformBidirectionalSubstringMatching() method attempts to find bidirectional substring matches between an item's (MaximoData) manufacturer numbers and a set of manufacturer numbers (manufacturerNums). Heres a concise summary:

  • Loop Through Manufacturer Numbers:

    • Iterates through each manufacturer number in manufacturerNums.

  • Bidirectional Substring Check:

    • For CXMANUFACTURERNUM and CXMANTYPED fields:
      • Checks if the field contains the manufacturer number or vice versa (bidirectional substring matching).
      • Ensures the match length is greater than 5 to filter meaningful matches.
      • If a match is found, sets isBidirectionalMatchValid to true.

  • Update Data Structures on Match:

    • If a bidirectional match is valid:
      • Assigns INDEX to the item using GetIndexFromTable().
      • If INDEX is not empty:
        • Adds the item to fuzzyMatchedList.
        • Marks the item as "Bidirectional" in matchedItems.
        • Adds the manufacturer number to matchedManufacturerNums.
        • Logs the match and exits the loop.
private void PerformBidirectionalSubstringMatching(MaximoData item, HashSet<string> manufacturerNums, Dictionary<MaximoData, string> matchedItems, HashSet<string> matchedManufacturerNums, List<MaximoData> fuzzyMatchedList)
 {
     foreach (var manufacturerNum in manufacturerNums)
     {
         bool isBidirectionalMatchValid = false;

         // Check for bidirectional substring match between CXMANUFACTURERNUM and mdlTable columns
         if (item.CXMANUFACTURERNUM != null)
         {
             if ((item.CXMANUFACTURERNUM.Contains(manufacturerNum) && manufacturerNum.Length > 5) ||
                 (manufacturerNum.Contains(item.CXMANUFACTURERNUM) && item.CXMANUFACTURERNUM.Length > 5))
             {
                 isBidirectionalMatchValid = true; // Match found
             }
         }

         // Check for bidirectional substring match between CXMANTYPED and mdlTable columns
         if (item.CXMANTYPED != null)
         {
             if ((item.CXMANTYPED.Contains(manufacturerNum) && manufacturerNum.Length > 5) ||
                 (manufacturerNum.Contains(item.CXMANTYPED) && item.CXMANTYPED.Length > 5))
             {
                 isBidirectionalMatchValid = true; // Match found
             }
         }

         if (isBidirectionalMatchValid)
         {
             if (!matchedItems.ContainsKey(item))
             {
                 item.INDEX = GetIndexFromTable(manufacturerNum); // Assign index for bidirectional match
             }

             if (!string.IsNullOrEmpty(item.INDEX))
             {
                 fuzzyMatchedList.Add(item); // Add to fuzzy matched list
                 matchedItems[item] = "Bidirectional"; // Mark as bidirectional matched
                 matchedManufacturerNums.Add(manufacturerNum); // Add manufacturer number to matched list
                 Debug.WriteLine($"MATCHER: Bidirectional substring match found in MaximoData: {item.CXMANUFACTURERNUM} or {item.CXMANTYPED}, matched with MDL: {manufacturerNum}, assigned INDEX: {item.INDEX}");
                 break; // Exit loop as soon as a bidirectional match is found
             }
         }
     }
 }

PerformFuzzyMatching

The PerformFuzzyMatching() method performs fuzzy matching between an item's (MaximoData) manufacturer numbers and a set of given manufacturer numbers (manufacturerNums). Here's a concise explanation:

  • Initialize Fuzzy Match Tracking:

    • bestFuzzyMatch: Stores the best matching manufacturer number.
    • highestSimilarity: Stores the highest similarity score found.

  • Calculate Fuzzy Similarity:

    • Iterates through each manufacturer number in manufacturerNums.
    • Uses Fuzz.TokenSetRatio (likely from the FuzzySharp library) to compute similarity scores between:
      • item.CXMANUFACTURERNUM and the manufacturer number.
      • item.CXMANTYPED and the manufacturer number.
    • Updates bestFuzzyMatch and highestSimilarity if a higher similarity score is found.

  • Mark Fuzzy Matches:

    • If the highest similarity score is above 96 (threshold) and bestFuzzyMatch is not null:
      • Assigns an INDEX to the item using GetIndexFromTable().
      • Ensures the item is not already in fuzzyMatchedList.
      • Adds the item to the fuzzyMatchedList and marks it as a fuzzy match (matchedItems).
      • Adds bestFuzzyMatch to the list of matched manufacturer numbers.
      • Logs the match information.
private void PerformFuzzyMatching(MaximoData item, HashSet<string> manufacturerNums, Dictionary<MaximoData, string> matchedItems, HashSet<string> matchedManufacturerNums, List<MaximoData> fuzzyMatchedList)
 {
     string? bestFuzzyMatch = null;
     double highestSimilarity = 0.0;

     // Calculate similarity scores using FuzzySharp for each manufacturer number
     foreach (var manufacturerNum in manufacturerNums)
     {
         double manufacturerNumSimilarity = item.CXMANUFACTURERNUM != null ? Fuzz.TokenSetRatio(item.CXMANUFACTURERNUM, manufacturerNum) : 0;
         double typeSimilarity = item.CXMANTYPED != null ? Fuzz.TokenSetRatio(item.CXMANTYPED, manufacturerNum) : 0;

         // Update best match based on highest similarity score
         if (manufacturerNumSimilarity > highestSimilarity)
         {
             highestSimilarity = manufacturerNumSimilarity;
             bestFuzzyMatch = manufacturerNum;
         }
         if (typeSimilarity > highestSimilarity)
         {
             highestSimilarity = typeSimilarity;
             bestFuzzyMatch = manufacturerNum;
         }
     }

     // If similarity is above threshold, mark as fuzzy matched
     if (highestSimilarity > 96 && bestFuzzyMatch != null)
     {
         if (!matchedItems.ContainsKey(item))
         {
             item.INDEX = GetIndexFromTable(bestFuzzyMatch); // Assign index for fuzzy match
         }

         if (!string.IsNullOrEmpty(item.INDEX) && !fuzzyMatchedList.Any(fuzzyItem => fuzzyItem.INDEX == item.INDEX))
         {
             fuzzyMatchedList.Add(item); // Add to fuzzy matched list
             matchedItems[item] = "Fuzzy"; // Mark as fuzzy matched
             matchedManufacturerNums.Add(bestFuzzyMatch); // Add manufacturer number to matched list
             Debug.WriteLine($"MATCHER: Fuzzy match found in MaximoData: {item.CXMANUFACTURERNUM} or {item.CXMANTYPED}, matched with MDL: {bestFuzzyMatch}, similarity: {highestSimilarity}, assigned INDEX: {item.INDEX}");
         }
     }
 }

private void FindUnmatchedManufacturerNumbers(HashSet<string> manufacturerNums, List<MaximoData> exactMatchedList, List<MaximoData> fuzzyMatchedList, List<Tuple<string, string>> unmatchedManufacturerNumsList, Dictionary<string, string> manufacturerNumToIndexMap)
